Kotlin中级（10）- - - Kotlin类之修饰符

发表于 2018-09-18 | 分类于 学习笔记 | | 阅读次数:

open和final

我们可以看到Person用open修饰了，证明它可以被继承，如果我们不用open关键字修饰，通过java字节码我们可以看出，原来的代码转化成了Java之后被默认加上了final，意味着不可被继承

多继承

在Kotlin中，我们在继承一个类后覆盖进行了override修饰，这个修饰也使这个属性默认open化

接口

在Kotlin中，接口本身和它内部的方法和属性都是默认加上open修饰符的，和普通类默认加上final修饰符石不同的
